# Минимальные подсказки для студентов

Проект представляет собой учебное Django-приложение для локального анализа незнакомой кодовой базы.

## Что нужно сделать

1. Запустите проект локально и создайте демонстрационные данные.
2. Восстановите структуру приложения: модели, маршруты, формы, шаблоны и основные пользовательские сценарии.
3. Проверьте поведение приложения для разных пользователей и ролей.
4. Найдите проблемы в коде, настройках и пользовательских сценариях.
5. Подготовьте исправления, не ломая существующую функциональность.
6. Добавьте или обновите тесты для подтверждения исправлений.
7. Оформите краткий отчет: что обнаружено, как воспроизводится, как исправлено.

## Быстрый запуск

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\\Scripts\\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_demo
python manage.py runserver
```

После запуска откройте приложение в браузере по адресу, который выведет Django.

## Направления анализа

Используйте этот список как ориентир, а не как готовый перечень ответов:

- права доступа и разделение ролей;
- обработка пользовательского ввода;
- формы и поля, доступные пользователю;
- работа с файлами;
- отображение данных в шаблонах;
- настройки проекта;
- обработка ошибок и граничных случаев;
- производительность запросов к базе;
- тесты и регрессионные сценарии.

## Важно

Работайте только в локальной учебной среде. Не публикуйте приложение в интернет и не используйте его как основу для production-проекта.
